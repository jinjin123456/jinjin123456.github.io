Event Bus实现 ​

  • VUe2中实现事件总线的常见做法就是定义一个bus对象(Vue实例)挂载在Vue的原型链上或者直接导出该bus对象都行

    • main.js中挂载

      js
      Vue.prototype.$bus = new Vue()
      js
      // A组件中触发
methods:{
	clickBtn(){
		this.$bus.$emit('clickBtn', '传值内容文本')
	}
}

// B组件中接收
created(){
	this.$bus.$on('clickBtn', res=>{
		console.log(res) // 传值内容文本
	})
}

    • 新建bus.ts文件，封装自定义事件总线文件，直接导出BUS对象

    js
    import Vue from "vue";
const BUS = new Vue();
export default BUS;
    js
    import BUS from './bus.ts'
// A组件中触发
methods:{
	clickBtn(){
		BUS.$emit('clickBtn', '传值内容文本')
	}
}

// B组件中接收
created(){
	BUS.$on('clickBtn', res=>{
		console.log(res) // 传值内容文本
	})
}

  • vue3中移除了移除了 $on 、 $off 和 $once 这几个事件 API，使得vue3无法像Vue2那样直接挂载一个Vue实例来实现EventBus。Vue官方推荐可以使用mitt和tiny-emitter两个第三方库来实现EventBus，官方示例是用的tiny-emitter ，我这里尝试用mitt实现：

    • 安装依赖

      js
      npm install mitt -S

    • 修改bus.ts文件，导出bus对象

      js
      import mitt from 'mitt'
const EventBus = mitt(
export default EventBus

    • 组件页面触发与接收

      JS
      import BUS from '@renderer/utils/bus'

// A组件中触发
methods:{
	clickBtn(){
		BUS.emit('test', [1,2,3,4,5])
	}
}

// B组件中接收
const handleBus = (data: any) => {
    console.log('handleBus', data)
}
onMounted(() => {
    BUS.off('test')
    BUS.on('test', handleBus)
})
onBeforeUnmount(() => {
    BUS.off('test') 
})

全局Loading组件 ​

  • 基于EventBus实现一个全局loading组件，在app任何页面开启/结束loading效果，并且可以配置loading文字，超时时间等等，可以处理多个接口连续不断loading的问题

    • 新建nLoading.vue，组件内处理isLoading事件总线的接收处理逻辑
    vue
    <script lang="tsx">
import { defineComponent, onMounted, onBeforeUnmount, Ref, ref } from 'vue'
import { Loading } from '@element-plus/icons-vue'
import BUS from '@renderer/utils/bus'

interface loadingMap {
  isLoading: boolean
  text?: string
  immediate?: boolean
  timeout?: number | undefined
}

export default defineComponent({
  name: 'NLoading',
  setup() {
    const loadingShow = ref(false)
    const loadingText = ref('加载中')
    const loadingTimer: Ref<null | ReturnType<typeof setTimeout>> = ref(null)
    const loadingHideTimer: Ref<null | ReturnType<typeof setTimeout>> = ref(null)
    const getLoadingConfig = ({
      immediate = false,
      timeout = 30,
      isLoading,
      text = '加载中'
    }: loadingMap) => {
      ...
    }
    onMounted(() => {
      BUS.off('isLoading')
      BUS.on('isLoading', getLoadingConfig) // getLoadingConfig方法主要用来处理接收到的loading状态，提示文字，以及异常处理等逻辑
    })
    onBeforeUnmount(() => {
      BUS.off('isLoading')
    })
    return {
      loadingShow,
      loadingText
    }
  },
  render() {
    return (
      <div class={`loading-mask ${this.loadingShow ? 'loading-show' : 'loading-hide'}`}>
        <div class="loading-box">
          <div class="lb-spinner">
            <el-icon class="is-loading">
              <Loading />
            </el-icon>
          </div>
          <div class="lb-info">{this.loadingText}...</div>
        </div>
      </div>
    )
  }
})
</script>

    • 任意组件页面实现对全局loading的控制
    js
    // A组件中模拟接口请求
const getApiInfo = () => {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            resolve()
        }, 5000) // 模拟等待接口返回信息
    })
}
// 点击按钮发送请求接口出现Loading效果
BUS.emit('isLoading', { isLoading: true, text: '正在loading中' })
getApiInfo().then((res) => {
    BUS.emit('isLoading', { isLoading: false })
    console.log(res)
})

小结 ​

  • 需要注意的是，在绝大多数情况下，不鼓励使用全局的事件总线在组件之间进行通信，短期内往往是最简单的解决方案，但从长期来看，不好追溯，可维护性很差。需根据项目具体情况选择适合的通信方案，这里只是作为一种方案的尝试和探索。
