在前端开发的过程中，我们经常会需要绑定一些持续触发的事件，如 resize、scroll、mousemove 等等，但有些时候我们并不希望在事件持续触发的过程中那么频繁地去执行函数。因为函数频繁调用、会存在大量计算导致页面卡顿的问题，或者重复请求接口之类的问题。通常这种情况下我们怎么去解决的呢？一般来讲，防抖和节流是比较好的解决方案。

所谓防抖，就是指触发事件后在 n 秒内函数只能执行一次，如果在 n 秒内又触发了事件，则会重新计算函数执行时间。将多次执行变为最后一次执行

  • 防抖：使用闭包存放setTimeout的id，当id不为空时，说明之前有触发事件，将其清除，重新setTimeout
js
// 参数为1.原函数 2.时间间隔
function debounce(func, wait) {
    let timeout;
    // 返回值是新函数
    return function (...args) {
        // 新函数内怎样去调用原函数,应保障两点
        // 两者接收的参数应当一致 args
        // 两者this指向应当一致 箭头函数
        clearTimeout(timeout);
        timeout = setTimeout(() => {
            func.apply(this, args);
        }, wait);
    }
}
/*
注1：setTimeout回调中使用arguments：
若不是箭头函数，接收的是回调的实际参数，而不是新函数的实际参数。
若是箭头函数，由于箭头函数没有arguments对象，接收的就是新函数的实际参数。
注2：setTimeout回调中使用this：
若是箭头函数，由于箭头函数没有this，因此回调中的this是外层的this，
若不是箭头函数，回调中的this指向window。
*/

// 进阶版-针对防抖而言还要考虑是否立即执行一次，之后都按照原先逻辑走，第三个参数immediate
function debounce(func, wait,immediate = false) {
    let timeout;
    return function (...args) {
        timeout && clearTimeout(timeout);
        if (immediate) {
            let callNow = !timeout;
            timeout = setTimeout(() => {
                timeout = null;
            }, wait)
            callNow && func.apply(this, args);
		} else {
            timeout = setTimeout(() => {
                func.apply(this, args);
            }, wait);  
        }
    }
}
  • 常见应用场景
    • 搜索框搜索输入。只需用户最后一次输入完，再发送请求
    • 手机号、邮箱验证输入检测
    • 窗口大小resize。只需窗口调整完成后，计算窗口大小。防止重复渲染。

所谓节流，就是指连续触发事件但是在 n 秒中只执行一次函数。将多次执行变为在规定时间内只执行一次，节流会稀释函数的执行频率。

  • 节流：使用闭包存放setTimeout的id，当id不为空时，说明之前有触发的事件。则不再添加新的setTimeout
js
function throttle(fn, delay) {
    let timer = null;
    return function(...args) {
        if (!timer) {
            timer = setTimeout(() => {
                fn.apply(this, args);
                timer = null;
            }, delay)
        }
    }
}

/*
由于定时器处于异步的宏任务队列，js中队列的执行并不那么精确，因此我们也可以采用时间戳来更加精准的实现
*/
// 进阶版 - 时间戳优化
function throttled(fn, delay) {
    let timer = null
    let starttime = Date.now()
    return function (...args) {
        let curTime = Date.now() 
        let remaining = delay - (curTime - starttime)  
        clearTimeout(timer)
        if (remaining <= 0) {
            fn.apply(this, args)
            starttime = Date.now()
        } else {
            timer = setTimeout(fn, remaining);
        }
    }
}
  • 常见应用场景
    • 滚动加载，加载更多或滚到底部监听
    • 搜索框，搜索联想功能
