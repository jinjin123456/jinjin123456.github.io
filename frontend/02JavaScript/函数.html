函数 ​

函数创建的主要三种形式 ​

  • 函数声明
    • 函数提升
  • 函数表达式
    • 匿名函数
  • 箭头函数 - 针对那些没有自己的“上下文”，但在当前上下文中起作用的短代码的
    • 简洁特性
      • 带花括号 - (...args) => { body }需要显式返回return
      • 不带花括号 - (...args) => expression
    • 没有自己的this和arguments，无法作为构造函数
  • 少见的创建方式 - let func = new Function ([arg1, arg2, ...argN], functionBody)
    • 应用场景非常特殊，需要从服务器获取代码或者动态地从模板编译函数时才会使用
    • 使用 new Function 创建的函数，它的 [[Environment]] 指向全局词法环境，而不是函数当前的词法环境。因此，我们不能在 new Function 中直接使用外部变量。

函数基本特性 ​

  • 块级作用域、局部变量
  • 作为参数传给函数的值，会被复制到局部变量
  • 访问外部变量
    • 推荐主要使用局部变量和参数，外部变量辅助，帮助函数简洁易懂
  • 返回值，如果没有返回值，默认返回undefined

函数进阶特性 ​

  • 函数参数
  • Rest参数 - 函数定义时使用
    • function func(arg1, arg2, ...rest) {}
    • 过去不支持Rest语法时，可以借助类数组arguments获取所有参数
  • Spread参数 - 函数调用时使用
    • Math.max(1,4, ...arr)
    • Spread语法还可以用来
      • 合并数组 - const newArr = [1, ...arr1, 3, ...arr2]
      • 复制/过滤对象
        • const objCopy =
        • 剔除少数属性：const { name, ...tmp } = obj
        • 保留少数属性：let namePerson = (({ name }) => ({name}))(person)
  • 闭包
    • 记住其外部变量并可以访问这些变量 的函数
  • 函数对象
    • 内建属性
      • name - 函数名称
      • length - 返回函数入参的个数，常应用于多态性场景
    • 添加自定义属性
    • 命名函数表达式(NFE)
      • let sayHi = function func() {}
      • 特性
        • 允许函数在内部引用自己
        • 在函数外是不可见的
  • setTimeout&setInterval
    • 使用形式
      • let timerId = setTimeout(func|code, [delay], [arg1], [arg2], ...) & let timerId = setInterval(func|code, [delay], [arg1], [arg2], ...)
      • clearTimeout(timerId) & clearInterval(timerId)
    • 嵌套的setTimeout
      • 要比setInterval更加灵活， 更精确地设置两次执行之间的延时
    • 垃圾回收
    • 当一个函数传入 setInterval/setTimeout 时，将为其创建一个内部引用，并保存在调度程序中。这样，即使这个函数没有其他引用，也能防止垃圾回收器（GC）将其回收， 直到 clearInterval 被调用
    • 零延时的setTimeout
      • 历史遗留问题 - 浏览器会将 setTimeout 或 setInterval 的五层或更多层嵌套调用（调用五次之后）的最小延时限制在 4ms。
    • 注意： 所有的调度方法都不能 保证 确切的延时
      • CPU 过载
      • 浏览器页签处于后台模式
      • 笔记本电脑用的是省电模式
      • ...等等这些都可能会将定时器的最小计时器分辨率（最小延迟）增加到 300ms 甚至 1000ms，具体以浏览器及其设置为准
  • call/apply - 装饰器模式和转发
    • 装饰器 - 可以接收另一个函数并改变其行为的特殊函数
      • 如果原始函数有属性，则装饰后的函数将不再提供这些属性
      • 使用场景1 - 透明缓存
        • 对象方法的缓存装饰器 - 需要指定上下文
          • func.call(context, ...args)
          • func.apply(context, args)
          • 两者调用几乎是等效的，只有一个关于args的差别，apply只接受类数组，apply相较call可能会更快
      javascript
      function slow(x) {
  // 这里可能会有重负载的 CPU 密集型工作
  alert(`Called with ${x}`);
  return x;
}

function cachingDecorator(func) {
  let cache = new Map();

  return function(x) {
    if (cache.has(x)) {    // 如果缓存中有对应的结果
      return cache.get(x); // 从缓存中读取结果
    }

    let result = func(x);  // 否则就调用 func
    // let result = func.call(this, x) // 指定上下文
    // let result = func.call(this, ...arguments) // 接收多参数

    cache.set(x, result);  // 然后将结果缓存（记住）下来
    return result;
  };
}

slow = cachingDecorator(slow);
// worker.slow = cachingDecorator(worker.slow) 

alert( slow(1) ); // slow(1) 被缓存下来了，并返回结果
alert( "Again: " + slow(1) ); // 返回缓存中的 slow(1) 的结果

alert( slow(2) ); // slow(2) 被缓存下来了，并返回结果
alert( "Again: " + slow(2) ); // 返回缓存中的 slow(2) 的结果

  return func.apply(this, arguments)

  • 函数绑定
    • func.bind(context, ...args)返回函数的"绑定的(bound)变体"
      • 绑定this - 解决setTimeout中丢失"this"问题
        • 两种解决方案：包装器和func.bind(context),前者存在改变风险，后者绑定更加稳定
      • 绑定参数 - 部分(应用)函数(Partial fuctions)
        • 通过绑定先有函数的一些参数来创建一个新函数
        js
        function mul(a, b){
  return a * b
}
let tripple = mul.bind(null, 3)
alert(tripple(3)) // = mul(3,3) = 9
        • 在没有上下文情况下的partial，比如对象方法
        js
        function partial(func, ...argsBound) {
  return function(...args) {
    return func.call(this, ...argsBound, ...args)
  }
}
