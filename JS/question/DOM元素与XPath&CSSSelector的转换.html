DOM元素与XPath&CSSSelector的转换

On this page
Table of Contents for current page

需求目标 ​

用户在网页浏览拾取移动鼠标的过程中选中特定元素之后，借助document.elementFromPoint(x,y)获取到该元素。由于x, y是当时用户交互产生，无法作为下一次机器人运行再次找到该元素的唯一标识，所以必须在刚刚拾取完成已知元素的情况下拿到该元素的唯一标识属性并记录入库，这样才能保障之后机器人执行时在同一页面操作为同一元素。真实的客户环境常常包含IE浏览器环境，在此背景下调研xpath路径和cssSelector与DOM元素的相互转换。

测试地址：https://captcha.ruijie.com.cn/

方案一：xpath方案 ​

已知DOM元素获取xpath ​

搜索很多没有找到JS中可以根据已获取的DOM元素获取xpath的直接可用的API，还是要根据纯DOM元素导航结构自己组织获取该元素的xpath路径的方法。

js
function getPathTo(element) {
    if (element.id!=='')
        return 'id("'+element.id+'")';
    if (element===document.body)
        return element.tagName;

    var ix= 0;
    var siblings= element.parentNode.childNodes;
    for (var i= 0; i<siblings.length; i++) {
        var sibling= siblings[i];
        if (sibling===element)
            return getPathTo(element.parentNode)+'/'+element.tagName+'['+(ix+1)+']';
        if (sibling.nodeType===1 && sibling.tagName===element.tagName)
            ix++;
    }
}

已知xpath获取DOM元素 ​

根据已知xpath获取DOM元素JS中有直接可用的api即document.evaluate()， document.evaluate() 就是通过 xpath 获取 dom 节点。返回的结果通过 .iterateNext() 可以获得 dom 对象，每次调用会少一个，当没有时就返回空。

  • 返回匹配到的第一个 xpath 对应的 dom 节点
js
function getElementByXpath(path) {
  return document.evaluate(path, document, null, XPathResult.FIRST_ORDERED_NODE_TYPE, null).singleNodeValue;
}
// or
function getElementByXPath(xpath) {
  var result = document.evaluate(xpath, document, null, XPathResult.ANY_TYPE, null);
  return result.iterateNext()
}

console.log(getElementByXpath('//*[@id="app"]/div[1]/main/div[1]/div/div[1]/div/div/div/div[2]'))

  • 返回匹配到 xpath 的 dom 节点集合
js
function getElementsByXPath(xpath) {
  var result = document.evaluate(xpath, document, null, XPathResult.ANY_TYPE, null);
  let node;
  const nodes = []
  while(node = result.iterateNext()){
      nodes.push(node);
  }
  return nodes;
}

console.log(getElementsByXPath('//*[@id="app"]/div[1]/main/div[1]/div/div[1]/div/div/div/div[2]'))

转换结果 ​

分别在Chrome和IE11控制台上测试转换代码：

js
console.log(getPathTo(document.elementFromPoint(300, 400)))
getElementByXPath('id("app")/DIV[1]/ASIDE[1]/UL[1]/LI[2]/SECTION[1]/UL[1]/LI[6]/A[1]')
  • 非IE浏览器 - Chrome：xpath和元素的相互转换成功。

  • IE浏览器：获取xpath成功，但根据xpath获取元素失败，原因是IE不支持document.evaluate()。xpath方向走不通，尝试第二种方案。

方案二：querySelector方案 ​

类似xpath，JS中没找到可以根据已获取的DOM元素获取css selector的直接可用的API，还是要根据纯DOM元素自己获取该元素的css selector。

已知DOM元素获取CSS Selector ​

js
function getCSSSelector(element) {
  let path = [];
  while (element !== document.body) {
    let selector = element.tagName.toLowerCase();
    if (element.id) {
      selector += '#' + element.id;
    } else {
      let sibling = element;
      let index = 1;
      while (sibling.previousElementSibling) {
        sibling = sibling.previousElementSibling;
        if (sibling.tagName.toLowerCase() === selector) {
          index++;
        }
      }
      if (index !== 1) {
        selector += ':nth-of-type(' + index + ')';
      }
    }
    path.unshift(selector);
    element = element.parentElement;
  }
  return path.join(' > ');
}

已知CSS Selector获取DOM元素 ​

很简单，直接document.querySelector()可以根据已知的CSS Selector获取DOM元素。
