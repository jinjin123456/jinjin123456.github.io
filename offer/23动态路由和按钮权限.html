2. 权限路由生成函数：

typescript
export function generateAuthRoutes(d: Array<any>, parentRoutes: Array<any> = []) {
  let authRoutes: Array<any> = []
  generateRoutes(d, parentRoutes, authRoutes)
  return authRoutes
}

export function generateRoutes(d: Array<any>, parentRoutes: Array<any> = [], result: Array<any> = []) {
  return d.forEach((item: any) => {
    // 生成完整路径
    item.path = '/' + [...parentRoutes, item.route].join('/')
    
    if (item.children && item.children.length > 0) {
      // 有子路由的设置重定向到第一个子路由
      item.redirect = '/' + [...parentRoutes, item.route, item.children[0].route].join('/')
      result.push(item)
      generateRoutes(item.children, [...parentRoutes, item.route], result)
    } else {
      result.push(item)
    }
  })
}

D. 路由守卫拦截 ​

在 middleware/redirect.global.ts 中实现全局路由守卫：

typescript
export default defineNuxtRouteMiddleware((to) => {
  let targetPath = to.path.endsWith('/') ? to.path.slice(0, -1) : to.path
  
  const store = useStore();
  // 检查目标路径是否在用户有权限的路由中
  let isInauthRoute = store.authRoutes.find((i:dataMap) => i.path === targetPath)
  
  // 如果路径不存在但在权限路由中，且有重定向配置，则自动重定向
  if(!isInNuxtRoute && isInauthRoute && isInauthRoute.redirect){
    return navigateTo(isInauthRoute.redirect)
  }
})

3. 按钮权限实现 ​

A. 按钮权限数据生成 ​

在解析菜单权限时同时生成按钮权限：

typescript
const getRouteBread = (targetData: Array<dataMap>) => {
  const btnObj = {}
  
  function go (arr: Array<dataMap>, parent: IBreadcrumb) {
    arr.forEach((item: dataMap) => {
      if (item.type === 3) { // type = 3 表示按钮权限
        btnObj[item.pRoute || ''] = btnObj[item.pRoute || ''] || {}     
        btnObj[item.pRoute || ''][item.code] = true
      }
      // 递归处理子节点
      if (item.children?.length) {
        go(item.children, { route: getRoutePath([], item), name: item.name, type: item.type })
      }
    })
  }
  
  go(targetData, {})
  // 将按钮权限存储到全局状态
  useStore().setBtnPermission(btnObj)
  return obj
}

B. 按钮权限检查函数 ​

typescript
export function btnPermission(permission: (string | number)[]) {
  return permission 
    && useStore().btnPermission 
    && useStore().btnPermission[permission[0]] 
    && useStore().btnPermission[permission[0]][permission[1]]
}

权限检查逻辑：

  • permission[0]：页面模块标识（如 'task', 'deptEmployee'）
  • permission[1]：具体按钮权限码（如 'elc_center_schedule_task_add'）

5. 权限系统特点 ​

A. 设计优势 ​

  1. 细粒度控制：支持到按钮级别的权限控制
  2. 动态性：权限数据完全由后端控制，前端动态生成
  3. 缓存机制：权限数据持久化存储，减少重复请求
  4. 组件化：权限检查逻辑高度封装，使用简单

B. 权限控制流程 ​

用户登录 → 获取权限树 → 解析权限数据 → 生成路由+按钮权限 → 
路由守卫验证 → 组件权限检查 → 显示/隐藏功能

C. 权限更新机制 ​

  1. 登录时更新：用户登录后重新获取最新权限
  2. 自动登录：支持从客户端自动登录并更新权限
  3. 持久化存储：权限数据存储在 localStorage，刷新页面不丢失

总结 ​

这个项目的权限系统实现了RBAC（基于角色的访问控制）模型，通过以下机制确保系统安全：

  1. 后端权限树：后端返回用户有权限的完整菜单和按钮树
  2. 前端解析：前端解析权限树，生成路由权限和按钮权限映射
  3. 路由守卫：全局中间件拦截无权限路由访问
  4. 组件级控制：在组件中通过 btnPermission 函数控制按钮显示/隐藏
  5. 状态管理：使用 Pinia 集中管理权限状态并持久化

这种设计既保证了安全性，又提供了良好的用户体验和开发体验。
